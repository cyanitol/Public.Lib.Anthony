## What Was Implemented

### ✅ IN (SELECT ...)

```sql
SELECT * FROM users WHERE id IN (SELECT user_id FROM orders)
SELECT * FROM products WHERE category NOT IN (SELECT name FROM disabled_categories)
```

**Bytecode Pattern:**
- Opens ephemeral table for subquery results
- Iterates through results comparing with LHS value
- Returns true/false based on match
- Supports NOT IN variant

### ✅ Scalar Subqueries

```sql
SELECT name, (SELECT MAX(salary) FROM employees) FROM users
SELECT * FROM orders WHERE amount > (SELECT AVG(amount) FROM orders)
```

**Bytecode Pattern:**
- Initializes result to NULL
- Uses OpOnce for single execution
- Reads one value from subquery result
- Enforces single-row constraint (error if multiple rows)

### ✅ EXISTS (SELECT ...)

```sql
SELECT * FROM users WHERE EXISTS (SELECT 1 FROM orders WHERE orders.user_id = users.id)
SELECT * FROM products WHERE NOT EXISTS (SELECT 1 FROM inventory WHERE inventory.product_id = products.id)
```

**Bytecode Pattern:**
- Initializes result to false
- Short-circuits after finding first row
- Returns true if any row exists
- Supports NOT EXISTS variant

## VDBE Opcodes Reference

| Opcode | Purpose in Subqueries |
|--------|----------------------|
| OpInteger | Initialize result to 0/1, set true/false values |
| OpNull | Initialize scalar subquery result to NULL |
| OpOpenEphemeral | Create temporary table for subquery results |
| OpRewind | Start iterating through subquery results |
| OpNext | Advance to next row in result set |
| OpColumn | Extract column value from result |
| OpEq | Compare values for IN clause |
| OpIf / OpIfNot | Conditional branching based on match |
| OpGoto | Jump to end after match found |
| OpClose | Close ephemeral table cursor |
| OpNot | Negate result for NOT IN / NOT EXISTS |
| OpOnce | Ensure scalar subquery executes only once |
| OpHalt | Error exit for multiple row constraint |
| OpNoop | Placeholder/marker for future compilation |

## Examples of Generated Bytecode

### IN Subquery Bytecode Structure:
```
0: OpInteger(0, resultReg)              ; init to false
1: OpOpenEphemeral(cursor, 1)           ; open temp table
2: [SELECT compilation would go here]
3: OpRewind(cursor, end)                ; start scan
4: OpColumn(cursor, 0, valueReg)        ; read value
5: OpEq(exprReg, valueReg, cmpReg)      ; compare
6: OpIf(cmpReg, setTrue)                ; if match
7: OpNext(cursor, 4)                    ; next row
8: OpClose(cursor)                      ; cleanup
9: [OpNot for NOT IN]
```

### Scalar Subquery Bytecode Structure:
```
0: OpNull(0, resultReg)                 ; init to NULL
1: OpOnce(onceReg, skip)                ; run once
2: OpOpenEphemeral(cursor, 1)           ; open temp
3: [SELECT compilation would go here]
4: OpRewind(cursor, noRows)             ; check rows
5: OpColumn(cursor, 0, resultReg)       ; read value
6: OpNext(cursor, error)                ; check 2nd row
7: OpHalt("too many rows")              ; error exit
8: OpClose(cursor)                      ; cleanup
```

### EXISTS Bytecode Structure:
```
0: OpInteger(0, resultReg)              ; init to false
1: OpOpenEphemeral(cursor, 1)           ; open temp
2: [SELECT compilation would go here]
3: OpRewind(cursor, end)                ; check rows
4: OpInteger(1, resultReg)              ; found row, set true
5: OpClose(cursor)                      ; cleanup
6: [OpNot for NOT EXISTS]
```

## Common Pitfalls

1. **Don't forget NOT variants**: Both IN and EXISTS support NOT
2. **Scalar subquery constraint**: Must return exactly 0 or 1 row
3. **Resource cleanup**: Always close ephemeral table cursors
4. **NULL semantics**: IN with NULL has special three-valued logic
5. **Register allocation**: Ensure sufficient registers for nested expressions
