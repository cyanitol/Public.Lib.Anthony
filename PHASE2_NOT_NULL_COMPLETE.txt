╔══════════════════════════════════════════════════════════════════════════════╗
║                         VALIDATION FLOWS                                      ║
╚══════════════════════════════════════════════════════════════════════════════╝

INSERT VALIDATION FLOW:
  1. Parse INSERT statement → *parser.InsertStmt
  2. Look up table schema → *schema.Table
  3. Create validator → NotNullConstraint
  4. Build values map → map[string]interface{}
  5. Apply DEFAULT values → fills missing/NULL columns
  6. Validate NOT NULL → check all NOT NULL columns
  7. If validation fails → return error
  8. If validation passes → compile VDBE bytecode
  9. Execute INSERT

UPDATE VALIDATION FLOW:
  1. Parse UPDATE statement → *parser.UpdateStmt
  2. Look up table schema → *schema.Table
  3. Create validator → NotNullConstraint
  4. Build updates map → map[string]interface{}
  5. Validate NOT NULL → check updated columns only
  6. If validation fails → return error
  7. If validation passes → compile VDBE bytecode
  8. Execute UPDATE

NOT NULL is the SIMPLEST constraint because:
  • No expression evaluation needed (unlike CHECK)
  • No index lookups needed (unlike UNIQUE)
  • No cross-table references (unlike FK)
  • Just check: value exists AND value != NULL
