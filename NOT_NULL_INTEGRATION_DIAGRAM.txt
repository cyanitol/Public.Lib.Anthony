NOT NULL CONSTRAINT - INTEGRATION ARCHITECTURE
==============================================

┌─────────────────────────────────────────────────────────────────────────────┐
│                          SQL Statement Flow                                  │
└─────────────────────────────────────────────────────────────────────────────┘

   INSERT INTO users (name, email) VALUES ('John', NULL);
           │
           ├─ Parser (internal/parser)
           │  └─→ *parser.InsertStmt
           │
           ├─ Driver (internal/driver/stmt.go)
           │  └─→ compileInsert(vm, stmt, args)
           │
           ├─ Schema Lookup
           │  ├─ schema.GetTable("users")
           │  └─→ *schema.Table {
           │       Columns: [
           │         {Name: "id",    NotNull: true,  Default: nil},
           │         {Name: "name",  NotNull: true,  Default: nil},
           │         {Name: "email", NotNull: true,  Default: "unknown@example.com"},
           │       ]
           │     }
           │
           ├─ NEW: Constraint Validation
           │  │
           │  ├─ constraint.NewNotNullConstraint(table)
           │  │  └─→ *NotNullConstraint{table: table}
           │  │
           │  ├─ Build values map:
           │  │  values := map[string]interface{}{
           │  │    "name":  "John",
           │  │    "email": nil,
           │  │  }
           │  │
           │  ├─ Apply defaults:
           │  │  nnc.ApplyDefaults(values, true)
           │  │  └─→ values = map[string]interface{}{
           │  │        "name":  "John",
           │  │        "email": "unknown@example.com",  // ← DEFAULT applied
           │  │      }
           │  │
           │  └─ Validate NOT NULL:
           │     nnc.ValidateInsert(values)
           │     ├─ Check "id":    MISSING → ERROR
           │     ├─ Check "name":  "John"  → OK
           │     └─ Check "email": "unknown@example.com" → OK
           │
           ├─ If validation fails: return error
           │  └─→ "NOT NULL constraint failed: column id"
           │
           └─ If validation passes: compile VDBE bytecode
              └─→ OpInit, OpOpenWrite, OpNewRowid, OpMakeRecord, OpInsert, ...

User Input:
  INSERT INTO users (name, email) VALUES ('Alice', NULL);

Step 1: Parse Statement
  ┌─────────────────┐
  │  Parser         │
  │  ↓              │
  │ InsertStmt {    │
  │   Table: "users"│
  │   Columns: [    │
  │     "name",     │
  │     "email"     │
  │   ]             │
  │   Values: [     │
  │     ["Alice",   │
  │      NULL]      │
  │   ]             │
  │ }               │
  └─────────────────┘

Step 2: Schema Lookup
  ┌─────────────────────────────────┐
  │  schema.GetTable("users")       │
  │  ↓                               │
  │ Table {                          │
  │   Name: "users"                  │
  │   Columns: [                     │
  │     {Name: "id",    NotNull: T}  │
  │     {Name: "name",  NotNull: T}  │
  │     {Name: "email", NotNull: T,  │
  │      Default: "noreply@..."}     │
  │   ]                              │
  │ }                                │
  └─────────────────────────────────┘

Step 3: Build Values Map
  ┌─────────────────────────────────┐
  │ buildValuesMap(stmt, table)     │
  │ ↓                                │
  │ map[string]interface{}{          │
  │   "name":  "Alice",              │
  │   "email": nil,                  │
  │ }                                │
  └─────────────────────────────────┘

Step 4: Create Validator
  ┌──────────────────────────────────────┐
  │ nnc := NewNotNullConstraint(table)   │
  └──────────────────────────────────────┘

Step 5: Apply Defaults
  ┌──────────────────────────────────────┐
  │ nnc.ApplyDefaults(values, true)      │
  │ ↓                                     │
  │ For each column with Default:        │
  │   if values[col] == nil {            │
  │     values[col] = col.Default        │
  │   }                                   │
  │ ↓                                     │
  │ map[string]interface{}{               │
  │   "name":  "Alice",                   │
  │   "email": "noreply@...", // Changed!│
  │ }                                     │
  └──────────────────────────────────────┘

Step 6: Validate NOT NULL
  ┌──────────────────────────────────────┐
  │ nnc.ValidateInsert(values)           │
  │ ↓                                     │
  │ For each column with NotNull:        │
  │   if !exists(values[col]) {          │
  │     return error                     │
  │   }                                   │
  │   if values[col] == nil {            │
  │     return error                     │
  │   }                                   │
  │ ↓                                     │
  │ Check "id":    MISSING → ERROR!      │
  │                                       │
  │ ERROR: "NOT NULL constraint failed:  │
  │         column id"                   │
  └──────────────────────────────────────┘

  UPDATE users SET email = NULL WHERE id = 1;

Step 1: Parse Statement
  ┌──────────────────────┐
  │ UpdateStmt {         │
  │   Table: "users"     │
  │   Sets: [            │
  │     {Column: "email" │
  │      Value: NULL}    │
  │   ]                  │
  │   Where: ...         │
  │ }                    │
  └──────────────────────┘

Step 2: Build Updates Map
  ┌──────────────────────┐
  │ map[string]interface{│
  │   "email": nil       │
  │ }                    │
  └──────────────────────┘

Step 3: Validate NOT NULL
  ┌────────────────────────────────────┐
  │ nnc.ValidateUpdate(updates)        │
  │ ↓                                   │
  │ For each update:                   │
  │   col := table.GetColumn(colName)  │
  │   if col.NotNull && val == nil {   │
  │     return error                   │
  │   }                                 │
  │ ↓                                   │
  │ Check "email": NotNull=T, val=nil  │
  │                                     │
  │ ERROR: "NOT NULL constraint        │
  │         failed: column email"      │
  └────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                       Performance Profile                                    │
└─────────────────────────────────────────────────────────────────────────────┘

Time Complexity:
  • NewNotNullConstraint:  O(1)
  • ValidateInsert:        O(n) where n = number of columns
  • ValidateUpdate:        O(m) where m = number of updates
  • ApplyDefaults:         O(d) where d = columns with defaults
  • GetNotNullColumns:     O(n)
  • HasNotNullConstraint:  O(1) - hash lookup in schema

Space Complexity:
  • NotNullConstraint:     O(1) - single pointer
  • Values map:            O(n) - one entry per column
  • No additional allocations during validation

Expected Overhead:
  • INSERT: ~10-50μs for typical tables (5-20 columns)
  • UPDATE: ~5-20μs for typical updates (1-5 columns)
  • Negligible compared to VDBE compilation and execution
